# lore/MECHANICS/ROMANCE_RULES.md — 恋爱/攻略规则（卡带级，必须持久遵守）

RAG_HEAD:
- 覆盖范围：本卡带“可攻略/完全攻略/后宫与纯爱并存”的统一判定规则（跨战役一致）。
- 关键对象：Bond、RouteClear、RomanceLock、LeavePonyvilleGate。
- 读取优先级：当出现恋爱推进、告白、锁线、通行解锁判断时优先读取。
- 约束：不裁定剧情结果；只定义判定口径与应被记录的字段。

## 0) 定义
### 可攻略对象（Romanceable）
- 范围：**所有雌性小马**（含动画中戏份极少的背景雌性小马，可扩写命名）。
- 条件：必须在 `characters/NPCs/npc_roster.md` 登记 `@handle`，并有对应 `npc_*.md`（至少一句话人设）。

### 目标女主（Target）
- 当玩家明确“想攻略谁”时，DM 应把该对象记为当前 Target（运行态字段，不在卡带落盘）。

## 1) 指标与记录口径（不负责数值变化）
- Bond（0–5）：对当前 Target 的亲密度通用刻度。
- RouteClear（0+）：已“完全攻略”的路线数量计数。

## 2) 完全攻略（Route Clear）判定（统一口径）
某位女主的“完全攻略”满足以下三个条件（A+B+C）时成立：

A. **关系阈值**：Bond >= 5（对该女主）。

B. **承诺确认**：发生一次“承诺声明”事件（形式由战役决定），其最低要求是：
- 玩家明确表达“以她为重要对象并承担相应责任/边界”的意愿（不等于结局）。
- 女主在当下语境中给出“认可/接受/愿意继续”的明确态度。

C. **社会可持续**：玩家的身份与关系不会立刻造成系统性崩坏（DM 以 Heat/Rumor/Debt 做合理约束即可）。

一旦成立：
- DM 将 RouteClear +1。
- DM 可（可选）将该女主标记为 `RomanceLock=true` 或 `RomanceCommitted=true`（运行态字段）。

## 3) 后宫与纯爱如何并存（默认规则）
为兼容“后宫/纯爱/日常”，本卡带默认采用：
- **多线可推进**：允许对多位雌性小马推进 Bond（并行分线）。
- **纯爱承诺是‘锁线’**：当玩家对某位女主做出“承诺确认”（2.B）并选择“锁线”，DM 应限制对其他女主的同等强度推进（可继续日常友好，但避免同级恋爱承诺冲突）。
- **真结局/隐藏线**：由战役决定其额外门槛（例如：至少 1 条 RouteClear + 满足某些主线旗标），卡带不预设。

> 若你希望“严格纯爱”（从一开始就只能单线），应在具体战役中将其写入 `.DM_BLUEPRINT.md` 作为更严格的覆盖规则。

## 3.1) 允许反悔/分手（默认允许）
- 允许玩家撤回承诺或分手；但必须通过一次“明确沟通事件”完成（不默认一键切断）。
- 分手不自动回退 RouteClear（除非战役额外规定）；但会引发代价（例如 Rumor 或 Debt 上升、关系网重排），由 DM 依据场景裁定。

## 3.2) 偷偷开后宫与“被发现的惩罚”（卡带级口径）
- 允许玩家在未公开的情况下并行推进多线（偷偷后宫）。
- 若被发现（由 DM 判定触发条件：证据、目击、谣言链闭环等），必须产生可记录的惩罚结果，默认优先从以下选择 1–2 项：
  - Rumor +1（关系流言扩散）
  - Heat +1（被重点关注/被视为不可信）
  - Debt +1（人情债/赔礼/补偿承诺）
  - Bond -1（对被伤害对象）
- 惩罚不等于剧情失败，只是将“后宫行为”转化为可持续的博弈成本。

## 4) 离开小马谷门槛（与恋爱规则联动）
- LeavePonyvilleGate 默认锁定。
- 当 RouteClear >= 1：LeavePonyvilleGate 解锁（允许自由离开小马谷并开启多城剧情）。

## 4.1) 默认阻断表现（当 LeavePonyvilleGate=locked）
- 默认理由：紫悦认为外出对“身份异常的雾凇”过于危险，建议先在小马谷完成安全评估与稳定关系网。
- 具体表现由 DM 选择（护送/劝阻/许可/临时危机等），但不应与“可解锁性（RouteClear>=1）”矛盾。

## 5) 需要被索引的新增 NPC（扩写指南）
新增可攻略雌性小马时，至少补齐：
- `characters/NPCs/npc_roster.md`：加入一行 + Handle Mapping + Entry Pointers
- `characters/NPCs/npc_<id>.md`：一句话概述 + 路线主题（框架）
